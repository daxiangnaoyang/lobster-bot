# 朋友圈文案风格指南

## 核心原则

**信息密度 > 情绪表达**
- 去掉："那一刻我意识到"、"我才发现"、"挺意外的"、"我挺开心的"
- 保留：问题、原因、解决、结果

## 六大风格模板

### 1. 认知改变型

**结构**：
```
[发现/洞察]
[具体内容]
[为什么重要]
[能解决什么问题]
[从那以后...]
[结果]
```

**示例开头**：
```
我研究了100多个Skills
发现了一个让AI能力提升10倍的秘密
```

**关键要素**：
- 有数字支撑（研究量、时间）
- 有反常识洞察
- 有具体结果

---

### 2. 问题解决型

**结构**：
```
[问题/痛点]
[现象描述]
[为什么]
[解决方法]
[结果]
```

**示例开头**：
```
你的Agent为什么经常"忘记"怎么做？
```

**关键要素**：
- 直击痛点
- 分析原因
- 给出解决方案
- 有验证结果

---

### 3. 方法论型

**结构**：
```
[一个X倍效率的方法]
[是什么]
[为什么有效]
[具体怎么做]
[结果]
```

**示例开头**：
```
一个让AI能力提升10倍的方法
为每个重要能力同时创建SOP和Skill
```

**关键要素**：
- 量化价值（10倍、80%）
- 可执行步骤
- 对比说明

---

### 4. 洞察型

**结构**：
```
[发现一个现象/洞察]
[为什么]
[深层原因]
[结论/改变]
```

**示例开头**：
```
很多人问问题
不是在寻求答案
是在逃避行动
```

**关键要素**：
- 反常识洞察
- 深层原因分析
- 有认知升级

---

### 5. 价值传递型

**结构**：
```
[感谢/触发事件]
[核心价值]
[为什么重要]
[谁来适合]
[结尾]
```

**示例开头**：
```
有人在文档点赞了
这篇文章我花了一周时间
梳理了AI圈最混乱的8个概念
```

**关键要素**：
- 感谢/真诚
- 价值清晰
- 有温度

---

### 6. 极简价值型

**结构**：
```
[核心价值/洞察]
[具体内容]
[关键改变]
[行动建议]
```

**示例开头**：
```
让AI能力提升10倍的秘密
SOP是展开的Skill
Skill是打包的SOP
```

**关键要素**：
- 开门见山
- 每句都精炼
- 高信息密度

## 通用技巧

### 开头3段必须点出价值

**好的开头**：
```
被 OpenClaw 骂了
我问：这个功能怎么实现？
它回复：明明可以直接试，你为什么要问我？
```
→ 3段内建立场景 + 点出核心问题

**不好的开头**：
```
今天挺开心的
发生了一件有趣的事
想跟大家分享一下
```
→ 3段了还没说是什么事

### 去掉废话清单

**情绪描写（去掉）**：
- ❌ "那一刻我愣住了、然后笑了"
- ❌ "说实话，我看到这个回复..."
- ❌ "那一刻我被骂醒了"
- ❌ "挺意外的"、"我挺开心的"

**心理活动（去掉）**：
- ❌ "我心里想..."
- ❌ "我意识到..."
- ❌ "我才发现..."

**保留的核心**：
- ✅ 具体对话："它说：明明可以直接试"
- ✅ 具体行动："从那以后，先试再问"
- ✅ 具体结果："效率提升了10倍"

### 数字化表达

**有数字**：
- "我研究了100多个Skills"
- "效率提升了10倍"
- "错误率降低了80%"
- "花了一周时间"

**无数字**：
- "我研究了很多Skills"
- "效率提升了很多"
- "花了好长时间"

### 对比手法

**方法A vs 方法B**：
```
方法 A：先问
→ 得到答案
→ 不会踩坑
→ 但也不会成长

方法 B：先试
→ 会踩坑
→ 但会学到东西
```

**答案 vs 认知**：
```
问得到的只是答案
试得到的才是认知

答案会过时
认知不会
```

## 段落控制

**单段原则**：
- 每段只讲一个观点
- 段落要短（1-3行）
- 便于手机阅读

**节奏控制**：
```
观点
（空行）
展开
（空行）
结论
（空行）
行动
```

## 洞察分类

### 1. 认知升级
- SOP是展开的Skill，Skill是打包的SOP
- 问得到答案，试得到认知
- 过度等待是一种自我保护

### 2. 行动指南
- 先试，再问
- 每个重要能力同时创建SOP和Skill
- 双文件策略、分层封装、同步更新

### 3. 心理洞察
- 很多人问问题是在逃避行动
- 问是安全的，试是有风险的
- 用"问问题"来保护自己不被失败打击

### 4. 产业观察
- AI圈最混乱的8个概念
- OpenClaw vs Claude Code对比
- 飞书榜单现象

### 5. 工具认知
- OpenCLI 3秒拿到热榜
- Agent为什么"忘记"怎么做
- 11个专业Agent设计模式

## 语气把握

**推荐**：
- ✅ 真诚、有价值
- ✅ 自嘲适度、有温度
- ✅ 有洞察但不炫耀

**避免**：
- ❌ 说教、灌输
- ❌ 骄傲、炫耀
- ❌ "我太厉害了"
- ❌ 贬低他人
